Nuestro editor **de instrumentación** permite a los usuarios de aplicaciones .NET implementar instrumentación personalizada desde la interfaz de usuario de New Relic sin necesidad de reiniciar la aplicación instrumentada. Este editor:

* Permite a los usuarios autorizados editar instrumentación personalizada desde la interfaz de usuario de New Relic en lugar de los archivos XML locales.
* Valida contra el esquema de instrumentación personalizado.
* Proporciona una opción para los clientes que no tienen acceso ni al código de la aplicación ni a los hosts en los que se ejecutan sus aplicaciones.

## Definir instrumentación personalizada \[#defining]

<img title="Ajustes de APM: instrumentación .NET" alt="Ajustes de APM: instrumentación .NET" src={apmDotnetInstrumentationEditor} />

<figcaption>
  **[one.newrelic.com > Todas las capacidades](https://one.newrelic.com/all-capabilities) > APM y servicios > (seleccione una aplicación .NET) > Configuración > Instrumentación:** El editor de **Instrumentación** proporciona una manera fácil de ver o editar su instrumentación personalizada en la IU. Este ejemplo XML muestra el nombre del ensamblado, el nombre de la clase y el nombre del método.
</figcaption>

<Callout variant="important"> El despliegue de instrumentos en vivo causa un breve período de mayor sobrecarga. Esto puede ralentizar notablemente las solicitudes de aplicación durante varios segundos. </Callout>

Para usar el editor **de instrumentación**:

1. Vaya a **[one.newrelic.com > Todas las capacidades](https://one.newrelic.com/all-capabilities) > APM**. Busca tu app y haz clic en **Configuración > Instrumentación**.
2. Utilice el editor de **instrumentación** para proporcionar XML utilizando la misma sintaxis que se describe para [editar archivos .xml](/docs/agents/net-agent/custom-instrumentation/create-transactions-xml-net) locales y [añadir detalles.](/docs/agents/net-agent/custom-instrumentation/add-detail-transactions-xml-net) Utilice cualquiera de estas opciones desde la IU:

   * Escriba el código XML directamente.
   * Copia el código y pégalo desde un portapapeles.
   * Haga clic en **Descargar Sample XML** para crear un archivo de instrumentación de ejemplo como punto de partida.
   * Sube un archivo .xml seleccionando **Elegir archivo**. Para ver un ejemplo, vea [Ejemplo MyInstrumentation.xml](/docs/agents/net-agent/custom-instrumentation/add-detail-transactions-xml-net#sample).
3. El editor **de instrumentación** valida su código XML mientras escribe. Para realizar la validación XML fuera del editor, seleccione **Descargar XSD** y utilice el archivo XSD con su herramienta de validación XML favorita para diagnosticar problemas.
4. Para implementar los cambios, seleccione **Enviar**.

## Ver resultados de instrumentación \[#view]

El agente .NET comenzará la nueva instrumentación en unos pocos [ciclos de cosecha](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#harvest-cycle) (normalmente unos minutos). La sección de **instrumentación actual** de la IU **de instrumentación** mostrará el estado de los cambios aplicados.

El XML del editor de **instrumentación** se combina con el XML que existe en los archivos de instrumentación locales en la máquina que ejecuta el agente. El agente busca periódicamente cambios en el XML final (combinado) y los capta dinámicamente, sin requerir un reinicio del agente o la aplicación.
