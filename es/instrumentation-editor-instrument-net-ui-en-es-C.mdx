Nuestro editor de **instrumentación** permite a los usuarios de aplicaciones .NET implementar instrumentación personalizada desde la interfaz de usuario de New Relic sin necesidad de reiniciar la aplicación instrumentada. Este editor:

* Permite a los usuarios autorizados editar la instrumentación personalizada desde la interfaz de usuario de New Relic en lugar de los archivos XML locales.
* Valida contra el esquema de instrumentación personalizado.
* Ofrece una opción a los clientes que no tienen acceso ni al código de la aplicación ni a los hosts en los que se ejecutan sus aplicaciones.

## Definir la instrumentación personalizada [#defining]

<img title="APM Settings: .NET instrumentation" alt="APM Settings: .NET instrumentation" src={apmDotnetInstrumentationEditor} />

<figcaption>
  **[one.newrelic.com > Todas las capacidades](https://one.newrelic.com/all-capabilities) > APM y servicios > (seleccione una aplicación .NET) > Configuración > Instrumentación:** El **editor de instrumentación** proporciona una forma sencilla de ver o editar su instrumentación personalizada en la interfaz de usuario. Este ejemplo XML muestra el nombre del ensamblado, el nombre de la clase y el nombre del método.
</figcaption>

<Callout variant="important"> El despliegue de instrumentación activa provoca un breve periodo de mayor sobrecarga. Esto puede ralentizar notablemente las solicitudes de aplicaciones durante varios segundos. </Callout>

Para utilizar el editor de **instrumentación**:

1. Vaya a **[one.newrelic.com > Todas las capacidades](https://one.newrelic.com/all-capabilities) > APM**. Busca tu aplicación y haz clic en **Configuración > Instrumentación**.
2. Utilice el editor de **instrumentación** para proporcionar XML utilizando la misma sintaxis descrita para [editar archivos de instrumentación .xml locales](/docs/agents/net-agent/custom-instrumentation/create-transactions-xml-net) y [añadir detalles de instrumentación.](/docs/agents/net-agent/custom-instrumentation/add-detail-transactions-xml-net) Utilice cualquiera de estas opciones desde la interfaz de usuario:

   * Escriba directamente el código XML.
   * Copie el código y péguelo desde el portapapeles.
   * Haga clic en **Descargar XML de muestra** para crear un archivo de instrumentación de muestra como punto de partida.
   * Cargue un archivo .xml seleccionando **Elegir archivo**. Para ver un ejemplo, consulte [Ejemplo MiInstrumentacion.xml](/docs/agents/net-agent/custom-instrumentation/add-detail-transactions-xml-net#sample).
3. El editor de **instrumentación** valida su código XML a medida que escribe. Para realizar la validación XML fuera del editor, seleccione **Descargar XSD** y utilice el archivo XSD con su herramienta de validación XML favorita para diagnosticar problemas.
4. Para desplegar los cambios, seleccione **Enviar**.

## Ver los resultados de la instrumentación [#view]

El agente .NET iniciará la nueva instrumentación en unos pocos [ciclos de recolección](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#harvest-cycle) (normalmente unos minutos). La sección **Instrumentación actual** de la interfaz de usuario **de Instrumentación** mostrará el estado de los cambios aplicados.

El XML del editor de **Instrumentación** se combina con el XML que existe en los ficheros locales de instrumentación en la máquina que ejecuta el agente. El agente busca periódicamente cambios en el XML final (combinado) y los recoge dinámicamente, sin necesidad de reiniciar el agente o la aplicación.
