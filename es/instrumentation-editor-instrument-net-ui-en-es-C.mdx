Nuestro editor de **instrumentación** permite a los usuarios de la aplicación.NET implementar instrumentación personalizada desde la interfaz de usuario de New Relic sin necesidad de reiniciar la aplicación instrumentada. Este editor:

* Permite a los usuarios autorizados editar la instrumentación personalizada desde la interfaz de usuario de New Relic en lugar de desde los archivos XML locales.
* Se valida con el esquema de instrumentación personalizado.
* Ofrece una opción para los clientes que no tienen acceso al código de la aplicación ni a los hosts en los que se ejecutan sus aplicaciones.

<Callout variant="important">
  La implementación de instrumentación en vivo provoca un breve período de mayor sobrecarga. Esto puede ralentizar notablemente las solicitudes de aplicaciones durante varios segundos.
</Callout>

Para usar el editor de **instrumentación**:

1. Vaya a **[one.newrelic.com > Todas las capacidades > APM](https://one.newrelic.com/all-capabilities)**. Busca tu aplicación y, a continuación, haz clic en **Configuración > Instrumentación.**
2. Utilice el editor de **instrumentación** para proporcionar XML con la misma sintaxis que se describe para [editar archivos de instrumentación.xml locales y [añadir](/docs/agents/net-agent/custom-instrumentation/add-detail-transactions-xml-net) detalles de instrumentación](/docs/agents/net-agent/custom-instrumentation/create-transactions-xml-net). Usa cualquiera de estas opciones de la interfaz de usuario:

   * Escriba el código XML directamente.
   * Copia el código y pégalo desde un portapapeles.
   * Haga clic en **Descargar XML de muestra** para crear un archivo de instrumentación de muestra como punto de partida.
   * Cargue un archivo.xml seleccionando **Elegir archivo**. Para ver un ejemplo, consulte [Example MyInstrumentation.xml](/docs/agents/net-agent/custom-instrumentation/add-detail-transactions-xml-net#sample).
3. El editor de **instrumentación** valida el código XML a medida que lo escribe. Para realizar la validación XML fuera del editor, seleccione **Descargar XSD** y utilice el archivo XSD con su herramienta de validación XML favorita para diagnosticar problemas.
4. Para implementar los cambios, selecciona **Enviar**.

## Ver los resultados de la instrumentación [#view]

El agente.NET iniciará la nueva instrumentación en unos pocos [ciclos de recolección](/docs/using-new-relic/welcome-new-relic/getting-started/glossary#harvest-cycle) (normalmente unos minutos). La sección **Instrumentación actual** de la interfaz de usuario de **instrumentación** mostrará el estado de los cambios aplicados.

El XML del editor de **instrumentación** se combina con el XML que existe en los archivos de instrumentación locales de la máquina que ejecuta el agente. El agente busca periódicamente los cambios en el XML final (combinado) y los recoge de forma dinámica, sin necesidad de reiniciar el agente o la aplicación.
