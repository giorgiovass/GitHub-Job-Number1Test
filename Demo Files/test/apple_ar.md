# ساعي البريد وطلبات العينات

> يتيح لك Braze توليد طلبات واجهة برمجة تطبيقات نموذجية لجميع نقاط النهاية من خلال مجموعتنا. تغطي هذه المقالة المرجعية مجموعة بريد البراز، وما هي، وكيفية إعداد واستخدام المجموعة، بالإضافة إلى كيفية تحرير وإرسال الطلبات.

## ما هو ساعي البريد؟

Postman هو أداة تحرير بصري مجانية لبناء واختبار طلبات واجهة برمجة التطبيقات (API). على عكس الطرق الأخرى للتفاعل مع واجهات برمجة التطبيقات (مثل استخدام cURL)، يتيح لك Postman تعديل طلبات API بسهولة، وعرض معلومات الرأس، وأكثر من ذلك بكثير. لدى Postman القدرة على حفظ مجموعات أو مكتبات من طلبات API الجاهزة النموذجية. لتسهيل تشغيل عملائنا باستخدام واجهة برمجة التطبيقات الخاصة بنا REST، أنشأنا مجموعة تحتوي على أمثلة جاهزة لجميع نقاط نهاية واجهات برمجة التطبيقات لدينا. هذا نص جديد ممتاز هنا يجب اكتشافه أيضا!!

شاهد مجموعة ساعي البريد لدينا بالنقر **الركض في ساعي البريد** في أمامنا [وثائقيات ساعي البريد](https://documenter.getpostman.com/view/4689407/SVYrsdsG?version=latest#intro) لنبدأ.

## استخدام مجموعة بريد البريز

إذا كان لديك حساب Postman (يمكنك تحميل أنواع macOS وWindows وLinux من [موقع ساعي البريد الإلكتروني][1])، يمكنك فتح توثيق ساعي البريد في تطبيقك الخاص بالنقر على البرتقالي **الركض في ساعي البريد** button. يمكنك حينها [خلق بيئة](#setting-up-your-postman-environment)، أو استخدم بيئة واجهة برمجة التطبيقات Braze REST كقالب لدينا، وقم بتحرير النموذج المتاح `البوست` و `احصل` طلبات تناسب احتياجاتك الخاصة.

### إعداد بيئة ساعي البريد الخاصة بك

{٪ خام ٪} تستخدم مجموعة بريد البراز متغيرا للقالب، `{{instance_url}}`، لاستبدال رابط واجهة برمجة التطبيقات REST الخاص بنسخة Braze الخاصة بك بالطلبات الجاهزة، و `{{api_key}}` متغير لمفتاح API الخاص بك. بدلا من تعديل جميع الطلبات يدويا في المجموعة، يمكنك إعداد هذا المتغير في بيئة Postman الخاصة بك. يمكنك إما اختيار بيئتنا المصممة (Braze REST API Environment Template) من القائمة المنسدلة واستبدال قيم المتغيرات بقيم المتغيرات الخاصة بك، أو يمكنك إعداد بيئتك الخاصة. {٪ endraw ٪}

لإعداد بيئتك الخاصة، قم بالخطوات التالية:

1. من **مساحات العمل** تبويب، اختر **البيئات**.
2. اضغط على **+** بالإضافة إلى زر لإنشاء بيئة جديدة.
3. أعط هذه البيئة اسما (على سبيل المثال، "Braze API Requests") وأضف مفاتيح ل `instance_url` و `api_key` مع قيم تتوافق مع \[Braze instance]\[7] و \[Braze REST API Key]\[8].
4. انقر **حفظ**.

{٪ ملاحظة تنبيه ٪} في `البوست` أجسام الطلب، `api_key` يجب أن يكون ذلك مغلفا بين علامات اقتباس: `"مثالي-API-KEY-CASE"`. في `احصل` الروابط، لا ينبغي أن تكون كذلك. لقد قدمنا لك هذا التنسيق بالفعل في هذه الوثائق `البوست` أجسام طلب، `احصل` عناوين URL، وقالب البيئة ل `مفتاح-API الخاص بك هنا`. {٪ endalert ٪}

! \[إضافة متغيرات لمفتاح API ورابط المثيل إلى بيئة Braze REST API في Postman.]\[3]

### باستخدام الطلبات المعدة مسبقا من المجموعة

بعد أن تقوم بتكوين بيئتك، يمكنك استخدام أي من الطلبات المبنية مسبقا في المجموعة كقالب لبناء طلبات API جديدة. لبدء استخدام أحد الطلبات الجاهزة، انقر عليه ضمن **المجموعات** قائمة من ساعي البريد. سيفتح هذا الطلب كعلامة تبويب جديدة في النافذة الرئيسية لتطبيق Postman.

بشكل عام، هناك نوعان من الطلبات التي تقبلها نقاط نهاية واجهة برمجة تطبيقات Braze - `احصل` و `البوست`. حسب ما هو `HTTP` بالطريقة التي تستخدمها نقطة النهاية، ستحتاج إلى تعديل الطلب الجاهز بشكل مختلف.

#### تعديل طلب منشور

عند تحرير `البوست` اطلب، افتح الطلب وانتقل إلى **الجسم** القسم في محرر الطلبات. لسهولة القراءة، اختر **الخام** زر الراديو لتنسيق `JSON` طلب الجسم.

! \[تبويب الجسم عند تحرير طلب تتبع مستخدم POST في Postman]\[4]

#### تعديل طلب GET

عند تحرير `احصل` طلب وتعديل المعلمات التي تم تمريرها في رابط الطلب. للقيام بذلك، اختر **بارامز** قم بالتبويب وتحرير أزواج المفتاح-القيمة في الحقول التي تظهر.

! \[تبويب المعلمات عند تحرير قائمة استعلام GET لطلبات عناوين البريد الإلكتروني التي تم إلغاء الاشتراك بها في Postman.]\[5]

### أرسل طلبك

بعد أن يصبح طلب واجهة برمجة التطبيقات جاهزا، اضغط **أرسلني**. يرسل الطلب وتظهر بيانات الاستجابة في قسم تحت محرر الطلبات. من هنا، يمكنك عرض البيانات الخام التي تم استرجاعها من واجهة برمجة تطبيقات Braze، ورؤية كود استجابة HTTP، ومعرفة المدة التي استغرقها الطلب، وعرض معلومات الرأس.

! \[مثال على بيانات استجابة الجسم من طلب POST بحالة 201 تم إنشاؤه وزمن استجابة 269 مللي ثانية.]\[6]
